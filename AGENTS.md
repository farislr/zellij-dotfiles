## OVERVIEW

Zellij user configuration repository with OS-specific configs for Linux and macOS.
Shared configuration is extracted, with OS-specific keybindings handled via `install.sh`.

## STRUCTURE

```text
./
├── config.shared.kdl    # Plugins, load_plugins, web_client, global options
├── config.Linux.kdl     # Linux keybindings (secondary_modifier "Alt")
├── config.Darwin.kdl    # macOS keybindings (secondary_modifier "Shift+Cmd")
├── install.sh           # OS-aware installer (symlinks appropriate config)
├── .gitignore           # Excludes backups, .serena/, generated config.kdl
└── .serena/             # Local Serena metadata (operational, not Zellij runtime)
```

## INSTALLATION

```bash
# Clone dotfiles repo
git clone https://github.com/YOUR_USERNAME/dotfiles.git ~/dotfiles

# Run installer (auto-detects OS)
cd ~/dotfiles/zellij && ./install.sh

# Or specify OS manually
./install.sh linux
./install.sh darwin
```

The installer:
1. Backs up existing config with timestamp
2. Symlinks `config.Linux.kdl` or `config.Darwin.kdl` → `~/.config/zellij/config.kdl`
3. Works on Linux and macOS

## CONVENTIONS

- **Config split**: Shared config in `config.shared.kdl`, OS-specific in `config.Linux.kdl` / `config.Darwin.kdl`
- **Modifier keys**: Linux uses `Alt`, macOS uses `Shift+Cmd` as secondary modifier
- **Install script**: Auto-detects OS, creates timestamped backup, symlinks appropriate config
- **Zellij reads**: `~/.config/zellij/config.kdl` (symlink to platform-specific config)
- **Do NOT edit**: `config.kdl` directly - it is a symlink generated by `install.sh`
